# Product

## Register

product

## Users

个人开发者与程序员。他们在本地电脑上使用 AI 辅助编程，需要一个高效、专注、信息密度高的工作界面。核心工作流：与 AI 对话 → 查看 diff → 审批变更 → 在终端中验证。他们重视速度、键盘操作、和视觉聚焦——不希望被装饰性元素分散注意力。

## Product Purpose

Helix 是一个 AI 驱动的桌面 IDE，通过 Hermes 推理网关连接多种 LLM。它的存在是为了让开发者用最少的认知开销完成"想法 → 代码"的闭环：对话即编程，diff 即审查，一键即部署。成功标准是用户感觉"比手动写代码更快，且始终知道 AI 在做什么"。

## Brand Personality

精准、克制、专业。三词概括：Sharp / Calm / Competent。

不张扬，不花哨，但在细节上让人感到"这工具懂我"。像一把精心打磨的工具刀——外观朴素，手感卓越。

## Anti-references

- **不要** VS Code 的信息过载（太多面板、太多按钮、太多颜色）
- **不要** ChatGPT 网页版的松散间距和大圆角（浪费屏幕空间）
- **不要** 渐变背景、彩色卡片、emoji 装饰等"玩具感"元素
- **不要** 硬编码的十六进制颜色（破坏暗色/亮色主题一致性）

## Design Principles

1. **信息密度优先** — 每一像素都要服务于内容。间距紧凑但不拥挤，让开发者在一屏内看到更多上下文。
2. **状态即反馈** — 用户始终知道 AI 在做什么（思考中/工具调用/diff/完成），通过微妙但清晰的状态指示器，而非弹窗或通知。
3. **键盘至上** — 所有操作都可通过键盘完成。方向键导航、回车确认、ESC 取消是基本要求。
4. **主题一致性** — 所有颜色必须通过 CSS 变量系统流转，暗色和亮色模式同样优秀。零硬编码色值。
5. **克制的动效** — 动画服务于理解（指示进度、引导注意力），而非装饰。尊重 prefers-reduced-motion。

## Accessibility & Inclusion

- WCAG AA 标准：正文对比度 ≥ 4.5:1，大字 ≥ 3:1
- 键盘完全可操作，focus-visible 状态清晰可见
- 支持 prefers-reduced-motion
- 中文界面，字号不小于 12px
